# Spring Petclinic CI/CD Platform Playbook

This repository provisions a small CI/CD platform for the Spring Petclinic application.

It uses Ansible to install:

- Jenkins and Docker on the build host
- Nexus Repository Manager on the shared-services host
- SonarQube and PostgreSQL on the shared-services host

The repository does not contain the Spring Petclinic application source code. Jenkins checks it out from:

```text
https://github.com/spring-projects/spring-petclinic.git
```

## Architecture

```text
Control/build host
  - Jenkins on port 8080
  - Docker engine
  - Runs CI/CD pipeline

Shared-services host
  - SonarQube on port 9000
  - Nexus on port 8081
  - PostgreSQL for SonarQube
```

Inventory is generated by `inventory.py`:

- `CONTROL_IP_ADDR` controls the local build host address.
- `TARGET_IP_ADDR` controls the remote shared-services host address.

## Provision

```bash
ansible-playbook infra.yml
```

Limit to one host group:

```bash
ansible-playbook infra.yml --limit build_servers
ansible-playbook infra.yml --limit shared_services
```

## CI/CD Demo

After provisioning, use `demo/Jenkinsfile.petclinic` to create a Jenkins Pipeline job.

The pipeline performs a realistic CI/CD flow:

```text
Checkout Spring Petclinic
  -> run tests
  -> run SonarQube analysis
  -> package the application
  -> build a Docker image
  -> deploy the application container
```

Full demo instructions are in `demo/ci-cd-usecase.md`.
