# Spring Petclinic CI/CD Platform Playbook

This repository provisions a small CI/CD platform for the Spring Petclinic application using Ansible.

It installs:

- Jenkins and Docker on the build host
- Nexus Repository Manager on the shared-services host
- SonarQube and PostgreSQL on the shared-services host

The repository does not contain the Spring Petclinic application source code. Jenkins checks it out from:

```text
https://github.com/spring-projects/spring-petclinic.git
```

## Architecture

```text
Control/build host
  - Jenkins on port 8080
  - Docker engine
  - Runs CI/CD pipeline

Shared-services host
  - SonarQube on port 9000
  - Nexus on port 8081
  - PostgreSQL for SonarQube
```

Inventory is generated by `inventory.py`:

- `CONTROL_IP_ADDR` controls the local build host address.
- `TARGET_IP_ADDR` controls the remote shared-services host address.

## Repository Layout

```text
infra.yml                  Main playbook
inventory.py               Dynamic inventory
group_vars/all/main.yml    Shared non-secret variables
group_vars/all/vault.yml   Local secret variables, ignored by Git
roles/common               Base packages and Java
roles/jenkins_docker       Docker and Jenkins service
roles/nexus                Nexus Repository Manager
roles/sonarqube            PostgreSQL and SonarQube
demo/                      CI/CD demo runbook and Jenkinsfile
docs/roles.md              Role-by-role explanation
```

## Prerequisites

Install required Ansible collections:

```bash
ansible-galaxy collection install -r collections/requirements.yml
```

Create the local secrets file:

```bash
cp group_vars/all/vault.example.yml group_vars/all/vault.yml
```

Edit `group_vars/all/vault.yml` and set real values. For real usage, encrypt it:

```bash
ansible-vault encrypt group_vars/all/vault.yml
```

Set the target host if you do not want to use the default in `inventory.py`:

```bash
export TARGET_IP_ADDR=<shared-services-host-ip>
export CONTROL_IP_ADDR=127.0.0.1
```

## Provisioning

```bash
ansible-playbook infra.yml
```

Limit to one host group:

```bash
ansible-playbook infra.yml --limit build_servers
ansible-playbook infra.yml --limit shared_services
```

Validate syntax without connecting to hosts:

```bash
ANSIBLE_LOCAL_TEMP=/tmp/ansible-local ansible-playbook --syntax-check infra.yml
```

## Validation

```bash
./demo/verify-services.sh
```

Manual checks:

```bash
systemctl status docker
systemctl status jenkins
curl -I http://127.0.0.1:8080
```

On the shared-services host:

```bash
sudo systemctl status postgresql
sudo systemctl status sonarqube
sudo systemctl status nexus
curl -I http://127.0.0.1:9000
curl -I http://127.0.0.1:8081
```

## CI/CD Demo

After provisioning, use `demo/Jenkinsfile.petclinic` to create a Jenkins Pipeline job.

The pipeline performs a realistic CI/CD flow:

```text
Checkout Spring Petclinic
  -> run tests
  -> run SonarQube analysis
  -> package the application
  -> build a Docker image
  -> deploy the application container
```

Full demo instructions are in `demo/ci-cd-usecase.md`.

For a role-by-role explanation, see `docs/roles.md`.
